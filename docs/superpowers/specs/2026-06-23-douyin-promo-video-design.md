## 2. 技术选型

### 2.1 评估过的技术

| 技术 | 定位 | 授权 | 优势 | 劣势 |
|------|------|------|------|------|
| **HyperFrames** (HeyGen) | HTML 原生视频渲染框架 | Apache 2.0 | 无构建步骤、Agent 友好、确定性渲染、GSAP 动效精细 | 仅渲染层，不含脚本/TTS/音乐 |
| **OpenMontage** (calesthio) | 端到端智能体视频制作系统 | AGPL-3.0 | 全流程自动化、内置 TTS/字幕/音乐/自审 | Python+Node 双栈、AGPL 传染性 |

### 2.2 选定方案: HyperFrames + OpenAI API

**选择理由**:
1. **授权友好**: Apache 2.0 无商用限制
2. **环境简单**: 仅需 Node.js 22+ + FFmpeg，无需 Python 双栈
3. **Agent 友好**: HTML 即视频，迭代快，适合 AI 编码代理操作
4. **动效精细**: GSAP 对产品截图的动画控制最精细
5. **脚本/TTS/字幕**可直接用 OpenAI API 处理，不需要 OpenMontage 完整 pipeline

**技术栈**:
```
├─ HyperFrames CLI (HTML+GSAP 渲染 MP4)
├─ Puppeteer (捕获产品截图)
├─ OpenAI TTS API (yunwu.ai 生成中文旁白)
├─ FFmpeg (音频混合 + 最终编码)
└─ GSAP (动画引擎)
```

**环境要求**:
- Node.js 22+
- FFmpeg
- OpenAI 兼容 API (yunwu.ai)

## 3. 视频设计

### 3.1 视频概念

**主题**: 「期末复习，从崩溃到通关」
**核心信息**: KnowledgeFlow 用 AI 帮你从零开始，一个知识点一个知识点地搞定期末复习
**情感曲线**: 挫败 → 希望 → 掌控 → 成就

### 3.2 视频结构（38 秒，竖屏 9:16）

| 时间 | 场景 | 旁白/字幕 | 视觉 |
|------|------|-----------|------|
| 0-3s | 钩子 | "期末复习，看了一晚上还是啥也不会？" | 黑底白字大字 + 问号动画 + 时钟滴答声 |
| 3-6s | 痛点 | "教科书太厚、知识点跳着讲、复习没计划" | 模糊教科书堆 + 红色叉号 + 倒计时 |
| 6-10s | 引出 | "KnowledgeFlow - AI 帮你从零搞定期末" | 产品 Logo + 品牌色渐入 + 标语 |
| 10-16s | 功能1 | "智能学习计划：输入考试科目，AI 生成复习路线" | 学习计划页截图 + 路线图绘制动画 |
| 16-22s | 功能2 | "AI 知识点讲解：从前置概念开始，不跳步" | 学习页截图 + 高亮动画 + 知识点弹出 |
| 22-28s | 功能3 | "智能练习：即时反馈，错题深度分析" | 练习页截图 + 对勾动画 + 错题卡片 |
| 28-33s | 功能4 | "学习统计：进度可视，成就解锁" | 统计页截图 + 数字滚动 + 徽章弹出 |
| 33-38s | CTA | "让知识轻松流入你的脑海" + 网址 + 话题 | 品牌色背景 + 网址 + #vibecoding大赏 #traeai创造力大赛 |

### 3.3 视觉风格

- **画幅**: 1080×1920（9:16 竖屏，抖音标准）
- **配色**: 跟随产品品牌色
  - 主色: 蓝色系（#3B82F6 系列）
  - 背景: 深色（#0F172A）与浅色（#FFFFFF）交替
  - 强调色: 黄色（#FBBF24）用于关键词高亮
- **字幕**: TikTok 风格
  - 位置: 屏幕中下部（避开抖音 UI 区域）
  - 字体: 大号粗体，白色描边
  - 效果: 逐字显示，关键词放大+变色
- **动效**: GSAP 驱动
  - 截图入场: 淡入 + 缩放（0.8→1.0）
  - 文字入场: 从下滑入 + 淡入
  - 转场: 0.3s 淡出淡入
  - 强调: 脉冲缩放（1.0→1.05→1.0）

### 3.4 音频设计

- **旁白**: 中文女声（OpenAI TTS，tts-1 模型，nova 音色）
- **背景音乐**: 轻快积极、BPM 100-120
  - 前半段（痛点）: 略低沉
  - 后半段（解决方案）: 明快向上
- **音效**:
  - 0-3s: 时钟滴答声（营造紧迫感）
  - 功能切换: 轻柔"叮"声
  - CTA: 上扬音效

## 4. 实施步骤

### 4.1 环境准备

1. 验证 Node.js 22+ 和 FFmpeg 已安装
2. 安装 HyperFrames CLI: `npx hyperframes init video-project`
3. 配置 OpenAI API 环境变量（.env 文件，不入 Git）

### 4.2 素材准备

1. **产品截图**: 用 Puppeteer 从部署网站捕获 4 个核心页面
   - 学习计划页（/plans）
   - 学习页（/study）
   - 练习页（/practice）
   - 统计页（首页 stats 区域）
2. **品牌素材**: 提取产品 Logo、品牌色值
3. **背景音乐**: 获取免版税音乐（YouTube 音频库 / Pixabay Music）

### 4.3 脚本与音频

1. 基于视频结构撰写旁白脚本（中文，口语化，每秒约 4-5 字）
2. 调用 OpenAI TTS API 生成旁白音频
3. 获取背景音乐并调整时长

### 4.4 HTML 组合编写

1. 创建 HyperFrames 项目结构
2. 编写 `index.html`:
   - 定义 stage（1080×1920）
   - 8 个场景片段（data-start/data-duration）
   - 每个场景: 背景 + 截图/文字 + GSAP 动画
3. 编写 GSAP timeline:
   - 每个场景的入场/出场动画
   - 字幕逐字显示效果
4. 添加音频轨道（旁白 + 背景音乐）

### 4.5 渲染与迭代

1. `npx hyperframes preview` 浏览器预览
2. 检查动画时序、字幕同步、音量平衡
3. `npx hyperframes render` 渲染 MP4
4. 验证输出: 时长、分辨率、音视频同步

### 4.6 质量检查

- [ ] 时长 35-40 秒
- [ ] 分辨率 1080×1920
- [ ] 4 个核心功能展示清晰
- [ ] 逐字字幕同步准确
- [ ] 背景音乐与节奏匹配
- [ ] 前 3 秒有强钩子
- [ ] 结尾有 CTA + 抖音话题
- [ ] 音量适中（-14 LUFS 左右）

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| HyperFrames 渲染失败 | 阻断 | 先用简单组合验证环境，逐步增加复杂度 |
| TTS 音色不自然 | 降质 | 备选: 豆包 TTS / Edge TTS |
| 截图质量不足 | 降质 | 使用高 DPI 截图（deviceScaleFactor: 2） |
| 字幕同步不准 | 降质 | 基于 TTS 时间戳生成字幕，非固定时长 |
| 抖音压缩导致画质下降 | 降质 | 渲染高码率版本（10Mbps+） |
